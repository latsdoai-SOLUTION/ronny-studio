# רוני סטודיו — האפליקציה 🎬

מרכז השליטה של רוני ל-UGC. הכל במקום אחד: היום שלי · לידים · פניות מוכנות · אולפן הפקה.
נבנה ב-Next.js (עברית RTL). **רץ מקומית בלי שום מסד נתונים** — הנתונים בקבצי JSON בתיקיית `data/`.

> ✅ הקוד נבדק ונבנה בהצלחה (`next build` עבר נקי).

---

## איך מריצים (פעם ראשונה)

צריך **Node.js 18+** מותקן (יש לך 22 — מעולה).

1. פותחים טרמינל בתוך התיקייה `ronny-studio-app`.
2. מתקינים: `npm install`
3. מריצים: `npm run dev`
4. פותחים בדפדפן: **http://localhost:3000**

זהו. האפליקציה עולה עם כל הלידים והפניות האמיתיים שכבר אספנו.

### דרך Cursor (מומלץ לעבודה איתי)
פותחים את התיקייה `ronny-studio-app` ב-Cursor → טרמינל (Ctrl+`) → `npm install` ואז `npm run dev`.
משם אפשר להמשיך לבנות שלבים 2–7 לפי `../תוכנית-בנייה/00_ארכיטקטורה-ותוכנית-עבודה.md`.

## מה כבר עובד (שלב 1 ✅)

- **היום שלי** — 3 פעולות ראשיות, 3 הזדמנויות להיום, סטטיסטיקות, כלל המנוע.
- **לידים** — לוח Kanban לפי סטטוס; שינוי סטטוס נשמר לקובץ (לחיצה אחת).
- **פניות מוכנות** — מייל + DM מוכנים עם כפתור העתקה.
- **אולפן הפקה** — מסביר את צינור ההפקה (כלי הפייתון הקיימים); העלאה מהאפליקציה בשלב 4.

## מה הלאה (לפי התוכנית)
שלב 2: גילוי לידים אוטומטי + מחולל פניות AI · שלב 3: בריף→חבילת ריל · שלב 4: העלאה→רינדור מהאפליקציה · שלב 5: העלאה לשרת + worker יומי.

## מבנה
```
ronny-studio-app/
├── data/               ← הנתונים (אפשר לערוך ידנית)
│   ├── leads.json      ← הלידים (לוח ה-Kanban)
│   ├── outreach.json   ← הפניות המוכנות
│   └── settings.json   ← קול מותג, נישות, חוקים
├── src/app/            ← המסכים (היום / לידים / פניות / אולפן)
├── src/components/     ← רכיבים (Sidebar, CopyButton, StatusSelect)
└── src/lib/store.js    ← שכבת הנתונים (קל להחליף ל-Postgres בעתיד)
```

## הערות
- **שכבת נתונים:** ב-v1 בחרנו JSON מקומי במכוון — אפס התקנה, רץ בכל מחשב מיד. בשלב האירוח (5) נחליף ל-Postgres מאחורי אותו `store.js`.
- **בידוד:** הכל בתיקיית RONNY INSTAGRAM בלבד.
